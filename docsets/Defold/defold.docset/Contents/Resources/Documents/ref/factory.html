factory.create()

make a factory create a new game object

The URL identifies which factory should create the game object. If the game object is created inside of the frame (e.g. from an update callback), the game object will be created instantly, but none of its component will be updated in the same frame. Properties defined in scripts in the created game object can be overridden through the properties-parameter below. See go.property for more information on script properties.

PARAMETERS

url - the factory that should create a game object (url)

[position] - the position of the new game object, the position of the game object containing the factory is used by default (vector3)

[rotation] - the rotation of the new game object, the rotation of the game object containing the factory is used by default (quat)

[properties] - the properties defined in a script attached to the new game object (table)

[scale] - the scale of the new game object (must be greater than 0), the scale of the game object containing the factory is used by default (number or vector3)

RETURN

the global id of the spawned game object (hash)

EXAMPLES

How to create a new game object: 

function init(self)
    self.my_created_object = factory.create("#factory", nil, nil, {my_value = 1})
    -- communicate with the object
    msg.post(msg.url(nil, self.my_created_object), "hello")
end

And then let the new game object have a script attached: 

go.property("my_value", 0)

function init(self)
    -- do something with self.my_value which is now one
end
