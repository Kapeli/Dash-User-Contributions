Description

Use the srcXOr keyword to combine an object's colors with the color underneath the object.

The ink property determines how an object's colors combine with the colors of the pixels underneath the object to determine how the object's color is displayed.

When the srcXOr transfer mode is used, LiveCode performs a bitXOr operation on each component of the object color with the corresponding component of the color under the object.

object component bitXOr background component

For example, suppose an object's color is 45,0,255, and the color of the pixels under the object is 20,45,100. If the srcXOr transfer mode is used, the object's displayed color is 57,45,210 (the decimal equivalent). In binary it is:

00101101, 00000000, 11111111
00010100, 00101101, 01100100 Logic XOR
========  ========  ========
00111001, 00101101, 10011011

The srcXOr mode can be used only on Unix and Windowssystems. On Mac OS systems, objects whose ink property is set to this mode appear as though their ink were set to reverse.

A list of all transfer modes can be found in the transfer mode glossary page for easy reference.
