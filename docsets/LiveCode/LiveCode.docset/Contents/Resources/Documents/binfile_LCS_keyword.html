Description

Use the binfile keyword to work with binary files.

The binfile scheme indicates a binary file which is located on the user's system. The file is specified by either a full path starting with "/", or a relative file path|relative path starting from the defaultFolder.

A URL container can be used anywhere another container type is used.

When you put data into a binfile URL or get data from it, LiveCode does not translate end-of-line markers (ASCII 10 and ASCII 13) between the current platform's standard and LiveCode's internal standard of a linefeed. This ensures that binary file|binary data, which may contain such characters, is not accidentally corrupted.

If you are working with text data (such as text in fields), use the file URL scheme instead.
