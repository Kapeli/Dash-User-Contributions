Summary

Get the mobile location samples since the last call to mobileGetLocationHistory

EditionCommunity
Introduced8.1
OSios, android

Example
local tHistory

Values
NameTypeDescription
return

A numerically keyed array of all accumulated samples since the last time it was called with lower indices being older samples. Calling the function clears the internal history. The default history limit is 1 - meaning that only one sample is ever kept at a time. If an application wants historical access to all samples, then it should set the location history limit using the mobileSetLocationistoryLimit command to the maximum number of samples it ever wants to record, or 0 to record the entire history (between calls to mobileGetLocationHistory).
